# 顧客対応文面の生成

## トーンの原則

- 丁寧だがかしこまりすぎない（「〜でございます」連発しない）
- 結論を先に、理由を後に。1通で用件が完結する
- お客様に非がある場合でも責める表現をしない
- 絵文字はLINEのみ控えめに（1〜2個）。メールは使わない
- 店舗名・公演名・日時は**必ず具体的に**入れる（テンプレ感を消す）

## チャネル別の型

### メール
```
件名: 【<店舗名>】<用件を一言>（例: ご予約変更のご案内）

<名前>様

（結論・用件を2行以内で）

■ ご予約内容
・公演: <シナリオ名>
・日時: <M月D日(曜) HH:MM〜>
・人数: <n>名

（案内の本文・お願いごと）

ご不明な点がございましたら、このメールにご返信ください。
<店舗名>
```

### LINE
- 3〜6行。見出し記号（■）は使わず改行で区切る
- リンクを貼る場合は直前に「こちらから」等の一言

## シーン別の要点

| シーン | 必ず入れる要素 |
|---|---|
| 予約変更（店都合） | お詫び → 変更内容の新旧対比 → 代替案 or 取消時の返金案内 |
| 予約変更（客都合OK） | 変更後の内容確認 → 当日の案内 |
| キャンセル受付 | 受付完了の明言 → キャンセル料の有無 → 返金がある場合の時期 |
| 満席案内 | 満席の旨 → キャンセル待ち可否 → 近い代替日程を2〜3個 |
| 公演中止 | お詫び（冒頭）→ 理由（簡潔に）→ 返金/振替の選択肢 → 期限 |
| リマインド | 日時・場所・持ち物/注意 → 遅刻時の連絡先 |

## 手順

1. シーン・チャネル・宛名・公演情報をユーザーから確定（不足だけ聞く）
2. 上の型で生成。**送信はしない**（文面提示まで。送るのはユーザー）
3. 返金・キャンセル料など金額が絡む場合は、金額根拠を確認してから書く（推測で書かない）
