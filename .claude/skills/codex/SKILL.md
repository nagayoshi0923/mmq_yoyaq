# Codex 向け指示書・プロンプト生成

役割分担: **設計・UI仕様・レビューは Claude、実装は Codex**。
このスキルの成果物は「Codex にそのままコピペできるプロンプト」1個。

## 手順

1. タスクの背景・対象ファイル・期待する完成形を会話とコードから確定させる
   （不明点が残っているならプロンプトを作る前にユーザーに確認する）
2. 改善タスク（リファクタ・デザイン統一・性能）の場合は `docs/IMPROVEMENT_HANDOFF.md` を読み、
   該当タスクの記載と整合させる。台帳にないタスクなら追記案も出す
3. 下の形式でプロンプトを生成する

## プロンプトに必ず含める内容

```
## 目的
（1〜3行。何のための変更か）

## 対象ファイル
（変更してよいファイルの明示。パスで列挙）

## やること
（具体的な手順。曖昧さを残さない）

## やらないこと
- 対象ファイル以外の変更（リファクタ・クリーンアップ含む）
- 機能の削除・無効化・コメントアウト
- 🔒 公演モーダル・公演カードの見た目変更
- border-l-4 のステータス色アクセント追加
- native confirm()/alert()（共通 ConfirmDialog = @/components/patterns/modal を使う）
- text-* / font-* / leading-* の Tailwind クラス追加

## 検証
- npm run verify がグリーンであること
- （タスク固有の確認手順）

## コミット
- 日本語 Conventional Commits（例: refactor: ○○を共通化）
- 1作業 = 1コミット。staging ブランチに直接コミット
```

## 注意

- プロンプトは1つのコードブロックにまとめ、ユーザーがそのままコピペできる形で出す
- 「やること」には受け入れ条件（何ができたら完了か）を必ず入れる
- 変更対象の現状コード片や参照すべき既存実装（例: 似た画面の書き方）をプロンプト内に引用すると Codex の精度が上がる
- DBマイグレーションを伴うタスクは Codex に任せず、分割してマイグレーション部分は Claude/ユーザー側で扱う
