# テスト観点チェック

**観点の正は `docs/templates/test-perspectives.md`。必ず先に読むこと。**
（/smoke と /review3 も同じファイルを参照している。新しい事故が起きたら template に追記）

## 手順

1. `docs/templates/test-perspectives.md` を読む
2. 対象（今回の変更 diff / 指定された機能）を特定
3. 観点集の各セクションについて「この変更に関係あるか」を判定:
   - **関係あり** → 具体的なテストケースに落とす（入力値・操作手順・期待結果）
   - 関係なし → スキップ（理由は不要）
4. 出力: 関係ありの観点だけの表

```
| 観点 | テストケース（操作手順） | 期待結果 | 確認方法 |
|---|---|---|---|
| 定員境界 | 残1席の公演に2ブラウザで同時予約 | 片方は満席エラー | staging実機 |
```

5. 機械的に検証できるもの（ユニットテスト追加・SQLでの整合確認）は
   「自動化候補」として分けて提示

## 使いどころ

- 実装完了直後の自己検証（コミット前）
- /review3 のビジネスロジック視点の深掘り
- 「このバグの兄弟がいないか」の水平展開チェック
