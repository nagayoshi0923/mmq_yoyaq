# セッション引き継ぎ

長いセッションでは AI 側から区切りを提案してよい（その際もこのスキルの形式で引き継ぎを準備してから呼びかける）。

## 手順

1. このセッションでやったこと・未完了のことを整理する
2. 関連する進行台帳があれば更新する:
   - 改善タスク → `docs/IMPROVEMENT_HANDOFF.md`
   - 切り分けた既存バグ → あとで対応バグの台帳（memory）
3. 未コミットの変更があれば、コミットするか退避するかをユーザーに確認する
4. 下の形式で出力する

## 出力形式

```
## 📋 セッション引き継ぎ

### 完了したこと
- （コミットハッシュ付きで列挙）

### 未完了・保留
- （何が・どこまで・なぜ止まっているか）

### 現在の状態
- ブランチ: staging（コミット/push 状況）
- DB: 未適用マイグレーションの有無
- ステージング確認待ちの項目

### ⚠️ 次セッションで注意すること
- （ハマりどころ・前提・依存関係）

---

### 次セッション用プロンプト（コピペ用）

（1つのコードブロックで。以下を含める:
 - 何の作業の続きか（1〜2行の背景）
 - 最初にやるべき具体的な1手
 - 読むべきファイル・台帳のパス
 - 完了条件）
```

## 注意

- 次セッション用プロンプトは**必ず**出す（共通ルール）
- プロンプトには「このファイルを読め」だけでなく、結論・決定事項を直接書き込む（次セッションが再調査しないで済むように）
