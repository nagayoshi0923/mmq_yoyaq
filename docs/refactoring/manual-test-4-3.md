## 第1弾: 削除操作の分割（コミット 8ef79605）

分割したコード: 削除確認ダイアログ／削除実行／直接削除。
分岐は5つ。**D-1 〜 D-5 を順に**実施してください（所要 約10分）。

### 準備
1. スケジュール管理ページを開く（管理画面 → スケジュール）
2. テスト用の通常公演を1つ作る:
   空きマスをクリック → 公演追加モーダル → シナリオ・GM・時間を適当に設定 → 保存
   （名前が分かりやすいシナリオ推奨。以後「テスト公演A」と呼ぶ）

### D-1: 予約ゼロの通常公演を削除（基本の削除）
| 手順 | 期待される結果 |
|---|---|
| ① テスト公演Aのマスを右クリック → 「公演を削除」 | 削除確認ダイアログが開く |
| ② ダイアログで削除を確定 | ダイアログが閉じ、**マスから公演が即座に消える** |
| ③ ページをリロード | **リロード後も消えたまま**（DBから本当に消えている） |
| ④ 該当マスをクリック → 履歴タブ（あれば） | 削除の履歴が記録されている |

### D-2: 有効な予約がある公演の削除は「拒否」される（最重要分岐）
| 手順 | 期待される結果 |
|---|---|
| ① テスト公演Bを新規作成（予約受付ON） | — |
| ② 予約サイト側からその公演に1件予約を入れる（自分のテスト顧客でOK） | 予約完了 |
| ③ スケジュール表に戻り、公演Bを右クリック → 「公演を削除」 → 確定 | **削除されない**。オレンジの警告トースト:「この公演には1件の有効な予約が紐付いているため削除できません」＋「代わりに『中止』機能を…」の案内 |
| ④ 画面と、リロード後の画面 | **公演Bは残ったまま**（予約も無事） |

### D-3: キャンセル済み予約「だけ」なら削除できる
| 手順 | 期待される結果 |
|---|---|
| ① D-2 の予約を予約一覧モーダルからキャンセルする | 予約ステータスがキャンセルに |
| ② 公演Bを右クリック → 「公演を削除」 → 確定 | **今度は削除に成功**（キャンセル済み予約は削除を妨げない） |
| ③ リロード | 消えたまま |

### D-4: 貸切公演の削除（公演は消え、申込は「キャンセル」で台帳に残る）
※ 2026-06-12 仕様変更: 申込の物理削除をやめ、キャンセル状態での保持に変更（顧客記録の保全）
| 手順 | 期待される結果 |
|---|---|
| ① 貸切申込→承認で貸切公演を1つ作る（消してよいもの！） | スケジュール表に貸切公演が表示される |
| ② その貸切公演を右クリック → 「公演を削除」 → 確定 | 公演がマスから消える（警告トーストは出ない） |
| ③ 貸切管理ページを開く | **申込は残っていて、ステータスが「キャンセル」になっている** |
| ④ 該当マスの更新履歴（公演追加→更新履歴タブ） | **「削除」の履歴が記録されている**（作成履歴だけでなく） |
| ⑤ リロード | 公演は消えたまま・申込はキャンセルのまま |

### D-5: 予約一覧からの「直接削除」（確認ダイアログを経ない別ルート）
| 手順 | 期待される結果 |
|---|---|
| ① 貸切公演をもう1つ用意し、参加者（予約）を表示できる状態にする | — |
| ② 公演の予約者バッジ等から**予約一覧モーダル**を開く | 予約一覧が表示される |
| ③ 貸切の参加者を**全員キャンセル**する | 「全員キャンセルしたのでイベントを削除しますか」相当の確認が出る |
| ④ 削除を選ぶ | モーダルが閉じ、公演がスケジュール表から消える |
| ⑤ リロード | 消えたまま |

### 異常系（任意・余裕があれば）
- D-6: ネットを切断（DevTools → Network → Offline）して D-1 を実行 →
  赤いエラートースト「公演の削除に失敗しました」が出て、ダイアログが閉じ、公演は残る

## 第2弾: 中止・復活操作の分割（コミット 168090be）

分割したコード: 中止確認ダイアログ／中止実行（キャンセルメール送信含む）／中止の取り消し。
**C-1 〜 C-4 を順に**実施（所要 約10分）。メール確認があるので自分のメールで予約しておくこと。

### C-1: 通常公演の中止（予約あり・キャンセルメールが飛ぶ最重要分岐）
| 手順 | 期待される結果 |
|---|---|
| ① テスト公演を作り、予約サイトから**自分のメールアドレスで**1件予約 | 予約完了メールが届く |
| ② スケジュール表で公演を右クリック → 「中止」系メニュー | 中止確認ダイアログ（理由入力欄つき）が開く |
| ③ 理由に「テスト中止です」と入力して確定 | 公演が**中止表示**（打ち消し線/グレー）になる。削除はされない |
| ④ 自分のメールを確認 | **キャンセルメールが届き、本文に「テスト中止です」が入っている** |
| ⑤ リロード | 中止表示のまま |

### C-2: 理由を空のまま中止（デフォルト文言）
| 手順 | 期待される結果 |
|---|---|
| ① 別のテスト公演＋自分のメールで予約 → 右クリック → 中止 → **理由を空欄のまま**確定 | 中止される |
| ② メールを確認 | 「誠に申し訳ございませんが、やむを得ない事情により公演を中止させていただく…」の**定型文**が入っている |

### C-3: 中止の取り消し（復活）
| 手順 | 期待される結果 |
|---|---|
| ① C-1 で中止した公演を右クリック → 「中止を取り消す」系メニュー | 公演が通常表示に戻る |
| ② リロード | 復活したまま（※取り消しメールは送られない仕様。予約はキャンセルされたままなので注意） |

### C-4: 貸切公演の中止
| 手順 | 期待される結果 |
|---|---|
| ① 消してよい貸切公演を1つ用意（申込→承認） | — |
| ② 右クリック → 中止 → 理由入力 → 確定 | 貸切公演が中止表示になり、**申込者にキャンセルメール**が飛ぶ |
| ③ 貸切管理ページ | 該当申込がキャンセル扱いになっている |
