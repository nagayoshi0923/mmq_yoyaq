### 2. 招待メールの有効期限切れ
**状態**: 修正済み（2025-12-04）  
**症状**: 招待メールのリンクが短期間で切れる  
**影響**: 
- スタッフがパスワード設定できない
- 再招待が必要

**修正内容**: 
- [x] OTP有効期限を24時間（86400秒）に延長
- [x] Management API経由での設定更新スクリプト作成

**修正ファイル**: `scripts/update-auth-config.mjs`

**残課題**:
- [ ] 期限切れ時の自動再送機能
- [x] スタッフ管理画面から「再招待」ボタン追加（実装済み）

---

### 3. 招待済みスタッフが自己登録した場合の紐付け問題
**状態**: 修正済み（2025-12-04）  
**症状**: 招待メール期限切れ後に自己登録すると、新しいuser_idで顧客として登録される  
**影響**: 
- staffテーブルと紐付かない
- ロールが `customer` になる

**修正内容**:
- [x] 自己登録時に `staff.email` でスタッフテーブルを検索
- [x] 見つかった場合は `staff.user_id` を自動更新して紐付け
- [x] `users.role` を `staff` に設定

**修正ファイル**: `src/contexts/AuthContext.tsx`

### 5. 複数タブ/ブラウザでのセッション競合
**状態**: 修正済み（2025-12-04）  
**症状**: 複数タブでログイン/ログアウトすると状態が不整合になる可能性  
**影響**: 予期しないログアウト、ロール変更

**修正内容**:
- [x] BroadcastChannel APIを使用したセッション同期を実装
- [x] ログアウト時の全タブ通知（SIGNED_OUTイベント）
- [x] ログイン時のセッションリフレッシュ（SIGNED_INイベント）
- [x] ロール変更時の同期（ROLE_CHANGEDイベント）

**修正ファイル**: `src/contexts/AuthContext.tsx`
